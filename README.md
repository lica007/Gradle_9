## Перечень автоматизируемых сценариев

### Навигация
1. Поиск курса через "Каталог курсов", вводя название курса в меню поиска 
    - Открыть главную страницу "Нетология"
    - Нажать "Каталог курсов"
    - В меню поиска ввести "Инженер по тестированию"
    - Выбрать курс из результатов поиска
2. Поиск курса через раздел "Программирование"
    - Открыть главную страницу "Нетология"
    - Выбрать раздел "Программирование"
    - Найти и выбрать "Инженер по тестированию"
3. Поиск курса через "Каталог курсов" нажатием на кнопку "Программирование"
    - Открыть главную страницу "Нетология"
    - Нажать "Каталог курсов"
    - Выбрать раздел "Программирование"
    - Найти и выбрать "Инженер по тестированию"

## Перечень используемых инструментов с обоснованием выбора
1. **Selenide** (Тестирование веб-приложения: проверка взаимодействия элементов интерфейса)
2. **Java** (Язык программирования)
3. **Gradle** (Система сборки)
4. **Page Object** (Паттерн, для удобства создания тестов)

## Перечень необходимых разрешений, данных и доступов
1. Разрешение на тестирование веб-приложения
2. Доступ к тестируемой версии приложения
3. Данные тестового пользователя

## Перечень и описание возможных рисков при автоматизации
1. Автотесты нужно постоянно обновлять чтобы они соответвовали изменениям в коде и веб-приложении 
2. Организация тестирования там где она не нужна 
3. Недостаточное тестовое покрытие, из-за чего можно не заметить баг
4. Слишком большое количество тестов, что делает их сложными в поддержке
5. Высокие финансовые затраты на разработку и поддержку автотестов


## Перечень необходимых специалистов для автоматизации
Для автоматизации нужен **инженеров по автоматизированному тестированию** или команда автотестировщиков
